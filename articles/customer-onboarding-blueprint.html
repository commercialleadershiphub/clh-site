Customer Onboarding Blueprint (30/60/90)

Owners, artifacts and risk log that set accounts up to win.

Problem

Kickoffs happen but value lands late. Handoffs are messy and the “first value” date is unknown.

What good looks like

  • Documented 30/60/90 with owners (Sales→CS→PS).
  • Data pack, enablement plan, and day‑30 value milestone.
  • Risk log with weekly updates.

Steps

  1. Kickoff: goals, team, comms, timeline.
  2. Enablement: roles, sessions, homework.
  3. Integration & data: owners, due dates, proof.
  4. Value moment: define and schedule it.

Artifacts

  • Runbook (one‑pager), risk log, enablement tracker, value checklist.

Metrics

  • Time‑to‑first‑value (TTFV).
  • On‑time tasks completion %.
  • Kickoff→adoption conversion.

Risks

  • “Custom” onboarding for every account → templatize and constrain.
  • Owner ambiguity → RACI in the doc header.
